## Background

現状、全バッチは手動実行。自動化することで arXiv新聞を完全自律運用にする。

現在のバッチ構成:

| 頻度 | スクリプト | 役割 |
|---|---|---|
| 日次 | `fetch_daily.py` | arXiv最新論文を取得・スコアリングして `data/YYYYMMDD.json` を生成 |
| 週次 | `recommend.py` | ratings + map.json からおすすめ論文を生成 |
| 月次 | `map.py` | 全論文をBERTopicでクラスタリングして `map.json` を再生成 |

バッチ実行後は Cloudflare Pages への再デプロイが必要（静的ビルドのため）。

**制約**:
- specter2_base（440MB）はローカルGPU/CPUが必要 → クラウドWorkerでは動かない
- デプロイは `wrangler pages deploy dist/` → Cloudflare API Token が必要
- arXiv は月〜金のみ更新（土日は skip）

## Spec

### 自動化方式: GitHub Actions + Modal

embedding の実行環境として **Modal** を採用する。

**構成**:
- GitHub Actions: スケジュール管理・git commit/push・CF Pages デプロイ
- Modal: specter2_base のロード＆推論（CPU、メモリ制約なし）

**Modal を選んだ理由**:
- GitHub Actions 標準ランナー（7GB RAM）では specter2_base の推論がギリギリ → OOM リスク
- Modal は CPU インスタンスをスペック指定で起動できる
- Modal Volume でモデルキャッシュを永続化 → 毎回 440MB ダウンロード不要
- コスト: 月 ~$0.30（無料枠 $30/月に対して余裕）

**コスト試算**（Modal GPU T4: ~$0.59/hour）:

| バッチ | 論文数 | 実行時間 | 月間回数 | 月間コスト |
|---|---|---|---|---|
| 日次 | ~300件 | ~1分 | 20回 | ~$0.20 |
| 週次 | ratings分 | ~1分 | 4回 | ~$0.04 |
| 月次 | ~10,000件 | ~15分 | 1回 | ~$0.15 |
| **合計** | | | | **~$0.39/月**（無料枠 $30 の 1/75） |

### ワークフロー設計

```
.github/workflows/
  daily.yml     # 月〜金 09:00 JST（00:00 UTC）
  weekly.yml    # 月曜 09:30 JST（00:30 UTC）
  monthly.yml   # 毎月1日 10:00 JST（01:00 UTC）
```

各 workflow の共通フロー:
1. `pip install modal` + `modal token set`
2. `modal run scripts/xxx_modal.py` → JSON 生成（Modal 側で embedding 実行）
3. `npm run build`
4. `npx wrangler pages deploy dist/`
5. 生成 JSON を `git commit & push`

### Modal 実装方針

`scripts/specter2.py` の `Specter2Encoder` を Modal 対応ラッパーで包む。

```
scripts/
  modal_app.py       # Modal app 定義・Volume・image 設定
  fetch_daily.py     # modal.Function として呼び出せるよう対応
  recommend.py       # 同上
  map.py             # 同上
```

- `@modal.function` で embedding 部分だけ Modal 上で実行
- モデルは `modal.Volume` にキャッシュ（初回のみダウンロード）
- ローカル実行も引き続き可能（`modal run` なしでも動く）

### 必要なシークレット

| シークレット | 用途 |
|---|---|
| `MODAL_TOKEN_ID` | modal CLI 認証 |
| `MODAL_TOKEN_SECRET` | modal CLI 認証 |

## Key Decisions

- Decision: embedding 実行に Modal GPU (T4) を採用。Rationale: GitHub Actions 標準ランナーの RAM 制約（7GB）を回避。M2 24GB で 20分かかる map:full（10,000件）を ~15分に短縮。月コスト ~$0.39 で無料枠（$30）の 1/75。
- Decision: GitHub Actions はオーケストレーションのみ担当。Rationale: スケジュール・シークレット管理・git操作は Actions が得意。重い計算は Modal に委譲。
- Decision: Modal Volume でモデルキャッシュを永続化。Rationale: specter2_base（440MB）を毎回ダウンロードすると遅い＆コスト増。
- Decision: バッチ結果（JSON）を git commit して push。Rationale: Astro の静的ビルドは data/ の JSON をビルド時に読むため、リポジトリに含める必要がある。
- Decision: 月〜金のみ日次実行。Rationale: arXiv が土日更新しないため無駄な実行を省く。
- Decision: ローカル実行との後方互換を維持。Rationale: `modal run` なしでも手動実行できるようにする。

## Notes

- GitHub Actions 無料枠: 2,000 分/月。Actions 側は軽量なので 100 分以内に収まる見込み
- `data/` は `.gitignore` に入れていない前提（現在コミット済みのため）
- デプロイ後に `wrangler pages deployment list` でステータス確認するステップを追加すると運用しやすい
- map:full（10,000件）を標準とする。M2 24GB で ~20分、Modal T4 で ~15分の見込み
