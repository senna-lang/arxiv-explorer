## Background

現在、日次論文（score順）と週次おすすめ（top 3クラスタ）が1カラムで縦に並ぶ。ユーザーは直接の興味範囲外から「少し遠いけど面白いかもしれない論文」を発見する手段がない。また20件の論文が一直線に並ぶUIは改善の余地がある。

- recommend.py は top 3クラスタ内の高マッチ論文のみ抽出（cosine sim >= 0.60）
- フロントエンドは max-width: 800px の1カラム構成
- データフロー: map.json (クラスタ) → recommend.py → recommendations.json → Astro

## Spec

1. **セレンディピティ論文の抽出** (`recommend.py` 拡張)
   - top 3クラスタの直下（ランク4-6位）からcentroidとの類似度が中程度（0.45-0.65）の論文を抽出
   - recommendations.json に `serendipity` / `serendipity_clusters` フィールドを追加

2. **2カラムUIレイアウト** (`[date].astro` 変更)
   - daily論文: `papers-section` に包み 10件ずつ区切り、max-width: 800px を維持
   - 下部: CSS Grid 2カラムで左=おすすめ、右=こんなのもどう？（モバイルは1カラム）

3. **SerendipitySection.astro** 新規作成
   - RecommendSection.astro をベースに緑系カラースキームで視覚的に区別
   - データソース: `data.serendipity` / `data.serendipity_clusters`

## Key Decisions

- Decision: セレンディピティのスコアバンドを [0.45, 0.65] に設定。Rationale: top 3クラスタのmin_match_score(0.60)と重なる上限で「近すぎず遠すぎない」隣接領域を狙う。
- Decision: ランク4-6クラスタを対象。Rationale: 直下クラスタは意味的に最も近接しており、真のランダムより発見価値が高い。
- Decision: RecommendationsData に optional フィールドとして追加。Rationale: 既存の推薦データとの後方互換性を保つ。
